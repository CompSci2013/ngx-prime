### Step 905.2: Component Registration in AppModule

All feature components are declared directly in `AppModule` rather than using separate feature modules with lazy loading. This approach:

1. Simplifies the module structure
2. Ensures all components are available immediately
3. Avoids complexity of managing multiple feature modules

In `src/app/app.module.ts`:

```typescript
// src/app/app.module.ts
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { BrowserAnimationsModule } from '@angular/platform-browser/animations';
import { HttpClientModule } from '@angular/common/http';

import { AppRoutingModule } from './app-routing.module';
import { AppComponent } from './app.component';
import { HomeComponent } from './features/home/home.component';
import { DiscoverComponent } from './features/discover/discover.component';
import { PanelPopoutComponent } from './features/panel-popout/panel-popout.component';
import { AutomobileComponent } from './features/automobile/automobile.component';

// Framework and UI modules
import { FrameworkModule } from './framework/framework.module';
import { PrimengModule } from './primeng.module';

@NgModule({
  declarations: [
    AppComponent,
    HomeComponent,
    DiscoverComponent,
    PanelPopoutComponent,
    AutomobileComponent
  ],
  imports: [
    BrowserModule,
    BrowserAnimationsModule,
    HttpClientModule,
    AppRoutingModule,
    FrameworkModule,
    PrimengModule
  ],
  bootstrap: [AppComponent]
})
export class AppModule {}
```

**Why no lazy loading?**

For this application, all features are tightly integrated and benefit from being eagerly loaded. The framework components and domain configurations are shared across features, making lazy loading less beneficial. The simplified approach reduces complexity while maintaining good performance for a medium-sized application.
