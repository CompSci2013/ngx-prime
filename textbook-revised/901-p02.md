### Step 901.3: Create the Home Component Template

Create `src/app/features/home/home.component.html`:

```html
<!-- src/app/features/home/home.component.html -->
<!-- VERSION 1 (Section 901) - Home page with domain navigation -->

<div class="home-container">
  <!-- Header -->
  <div class="header">
    <h1>Generic Prime Discovery Framework</h1>
    <p class="subtitle">Explore data across multiple domains</p>
  </div>

  <!-- Domains -->
  <div class="domains-section">
    <div class="domain-grid">
      <a routerLink="/discover" class="domain-card">
        <div class="card-icon">🚗</div>
        <h3>Automobiles</h3>
        <p>Browse and analyze vehicle data</p>
      </a>
      <a class="domain-card disabled">
        <div class="card-icon">🌾</div>
        <h3>Agriculture</h3>
        <p>Agricultural data analysis</p>
      </a>
      <a class="domain-card disabled">
        <div class="card-icon">⚛️</div>
        <h3>Physics</h3>
        <p>Physics research data</p>
      </a>
      <a class="domain-card disabled">
        <div class="card-icon">🧪</div>
        <h3>Chemistry</h3>
        <p>Chemical compound data</p>
      </a>
      <a class="domain-card disabled">
        <div class="card-icon">📐</div>
        <h3>Mathematics</h3>
        <p>Mathematical datasets</p>
      </a>
    </div>
  </div>

</div>
```

**Template structure explained:**

| Section | Purpose |
|---------|---------|
| `.header` | Application branding and tagline |
| `.domains-section` | Navigation cards to feature areas |
| `.domain-card.disabled` | Future domain placeholders (not yet implemented) |

**Note on `routerLink`:** The `routerLink` directive from `RouterModule` creates navigation links that work with Angular's router. Unlike regular `href` attributes, `routerLink` doesn't cause a full page reload — it performs client-side navigation. The active Automobiles card links to `/discover`, while other domains are marked as `disabled` for future expansion.
