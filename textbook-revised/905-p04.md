### Step 905.3: Understanding Route Parameters

The panel pop-out route uses parameters to identify what component to show:

```
/panel/:gridId/:panelId/:type
       │       │        │
       │       │        └── Component type (chart, picker, etc.)
       │       └── Unique panel identifier
       └── Source grid identifier (e.g., discover)
```

**Example URLs:**

| URL | Meaning |
|-----|---------|
| `/panel/discover/chart-manufacturer/chart` | Manufacturer chart from discover page |
| `/panel/discover/query-control/query-control` | Query control panel |
| `/panel/discover/basic-results-table/basic-results` | Results table |
| `/panel/discover/manufacturer-model-picker/picker` | Manufacturer-model picker |
| `/panel/discover/statistics-panel-2/statistics-2` | Statistics panel |

The PanelPopoutComponent extracts these parameters to:
1. Initialize BroadcastChannel with the right panel ID
2. Determine which content to render based on panel type
3. Set up communication with the main window
