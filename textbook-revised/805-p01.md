# 805: Inline Filters Component

**Status:** Superseded by QueryControlComponent

---

## Notice

This section describes an `InlineFiltersComponent` that was planned but not implemented. The functionality has been superseded by **QueryControlComponent** which provides expanded functionality.

The directory `src/app/framework/components/inline-filters/` exists as an empty placeholder in the codebase.

---

## Migration Guide

The QueryControlComponent (`src/app/framework/components/query-control/query-control.component.ts`) provides all planned inline filter functionality plus additional features:

### Original Planned Features (Now in QueryControlComponent)
- Display active filters as chips
- Click to edit filter
- Click to remove filter
- Sync with URL state

### Additional Features in QueryControlComponent
- Filter field dropdown for selecting filterable fields
- Multiselect dialog for multi-value filters
- Range dialog for min/max filters
- Active highlight chips (separate from filter chips)
- Full PrimeNG Dialog integration
- Keyboard navigation and accessibility features
- Comprehensive bug fix implementations

---

## QueryControlComponent Location

```
src/app/framework/components/query-control/query-control.component.ts
```

### Basic Usage

```html
<app-query-control
  [domainConfig]="domainConfig"
  (urlParamsChange)="onUrlParamsChange($event)"
  (clearAllFilters)="onClearAllFilters()">
</app-query-control>
```

### Key Interfaces

The component uses these internal interfaces:

```typescript
interface ActiveFilter {
  definition: FilterDefinition;
  values: (string | number)[];
  urlValue: string;
}
```

### Architecture

- **PrimeNG-First**: Uses PrimeNG Dialog, Dropdown, Checkbox directly
- **URL-First**: All state changes via UrlStateService
- **OnPush**: Requires manual change detection via ChangeDetectorRef
- **Domain-Agnostic**: Works with any domain configuration

The component is approximately 970 lines of TypeScript with comprehensive documentation and bug fix comments.
