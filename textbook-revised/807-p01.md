# 807: Column Manager Component

**Status:** Planned - Not Yet Implemented

---

## Notice

This section describes a `ColumnManagerComponent` that is planned but not yet implemented. The directory `src/app/framework/components/column-manager/` exists as an empty placeholder in the codebase.

---

## Planned Functionality

When implemented, the Column Manager Component would provide:

1. **Column visibility toggle**: Show/hide columns without losing data
2. **Column reordering**: Drag columns to customize display order
3. **Persistence**: Settings survive browser refresh via localStorage
4. **Domain-agnostic**: Works with any table configuration

### Planned Storage Key

Column preferences would be stored in localStorage using a domain-specific key:
```
vvroom_columns_{domainId}
```

---

## Current Alternatives

Column management may be handled differently in the current implementation:
- **DynamicResultsTableComponent** may handle some column management internally
- Table configurations in DomainConfig define available columns
