# 809: Dockview Statistics Panel Component

**Status:** Planned - Not Yet Implemented

---

## Notice

This section describes a `DockviewStatisticsPanelComponent` that is planned but not yet implemented. The directory `src/app/framework/components/dockview-statistics-panel/` exists as an empty placeholder in the codebase.

The application currently uses **StatisticsPanel2Component** for chart layout, which uses CDK drag-drop grid instead of Dockview.

---

## Planned Functionality

When implemented, the Dockview Statistics Panel would provide:

1. **Tabbed panels**: Multiple charts in the same space, switch via tabs
2. **Resizable splits**: Drag borders to adjust chart sizes
3. **Drag to rearrange**: Drag tabs to reorder or split panels
4. **Consistent dark theme**: Matches the application's visual style
5. **Layout serialization**: Save and restore panel arrangements

---

## Current Alternative

Use **StatisticsPanel2Component** (Section 808) for chart layout:

```html
<app-statistics-panel-2
  [domainConfig]="domainConfig"
  (chartPopOut)="onChartPopOut($event)"
  (chartClicked)="onChartClick($event)">
</app-statistics-panel-2>
```

StatisticsPanel2 provides:
- CDK horizontal drag-drop grid
- Chart reordering via drag and drop
- Pop-out support for individual charts
- Integration with ResourceManagementService

## Future Implementation Notes

Dockview integration requires:
1. Additional npm package installation: `npm install dockview`
2. Complex Angular-to-Dockview DOM bridging
3. DOM manipulation pattern for moving Angular components into Dockview containers

### The DOM Manipulation Pattern

Dockview is framework-agnostic and expects to manage its own DOM. The integration pattern:

1. Render Angular chart components in a hidden container
2. When Dockview creates a panel, move the chart element into the panel
3. When Dockview disposes a panel, move the chart back to the hidden container

This keeps Angular happy while letting Dockview manage the layout.
